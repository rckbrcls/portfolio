This portfolio now has a local MDX blog. Posts live in the same repository as the
site, which keeps writing, review, and publishing close to the work they are
meant to explain.

## Why keep the blog inside the portfolio

A separate documentation surface would have added more structure than this site
actually needs. The goal here is simpler: longer notes should sit next to the
projects, not outside them.

Standard markdown images work with normal repository assets:

![Editorial diagram describing the blog system.](/images/blog/launching-the-blog/diagram.svg)

Inline math also works in normal prose, like $f(x) = x^2 + 2x + 1$.

$$
\int_0^1 x^2 \, dx = \frac{1}{3}
$$

### Authoring notes

- write posts in `content/blog/<slug>.mdx`
- keep post assets in `public/images/blog/<slug>/`
- use frontmatter for title, summary, date, tags, draft, and optional cover image

Code blocks stay readable without pulling the site into a documentation visual
language:

```ts
export function publishPost(slug: string) {
  return `/blog/${slug}`;
}
```
